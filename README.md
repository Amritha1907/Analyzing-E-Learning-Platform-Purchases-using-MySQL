# Analyzing-E-Learning-Platform-Purchases-using-MySQL

🎯 Project Overview

This project focuses on analyzing purchase data from an e-learning platform using MySQL to extract meaningful business insights. The goal is to understand user purchasing behavior, identify revenue patterns, and support data-driven decision-making.
E-learning platforms generate large amounts of transactional data, and analyzing this data helps improve course offerings, pricing strategies, and user engagement.

🛠️ Tools & Technologies

MySQL – Data querying and analysis
SQL – Data manipulation and aggregation
MySQL Workbench – Query execution and database management

📂 Dataset Description

The dataset includes information related to:

User purchases
Course details
Transaction amounts
Purchase dates

This structured data allows analysis of customer behavior and platform performance.

🔍 Key Analysis Performed

Data extraction and cleaning using SQL
Aggregation of revenue and purchase metrics
Customer purchase behavior analysis
Identification of top-performing courses
Time-based trend analysis

📈 Business Questions Answered

Which courses generate the highest revenue?
What are the purchasing patterns of users?
Which time periods show peak sales activity?
Who are the most valuable customers?

💡 Key Insights

High-value courses contribute significantly to total revenue
Certain time periods show increased purchasing activity
Repeat users drive a major portion of sales
Pricing and course category impact purchase decisions

🚀 Project Workflow

Data Import into MySQL
Data Cleaning and Preparation
Writing SQL Queries for Analysis
Aggregation and Insight Generation
Business Interpretation
